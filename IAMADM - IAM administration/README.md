# IAMADM - IAM administration

Enterprise identity and access management administration for Dynatrace.

> **Recommended:** Import the JSON files from NOTEBOOKS/ into a Dynatrace tenant for the best experience. These notebooks contain interactive DQL queries that execute against your environment's data.

## Structure
- NOTEBOOKS/ — Dynatrace notebook JSON files
- PDFs/ — Printable versions of each notebook
- markdown/ — Markdown exports of the notebooks

## Notebook Lineup
1. [Governance Foundations](markdown/-IAMADM-01-governance-foundations.md) — IAM governance principles and strategy
2. [SSO & Authentication](markdown/-IAMADM-02-sso-authentication.md) — Single sign-on and authentication setup
3. [Group Architecture](markdown/-IAMADM-03-group-architecture.md) — Designing group structures
4. [Policy Authoring](markdown/-IAMADM-04-policy-authoring.md) — Writing and managing policies
5. [Boundary Design](markdown/-IAMADM-05-boundary-design.md) — Defining access boundaries
6. [User Lifecycle](markdown/-IAMADM-06-user-lifecycle.md) — Managing user provisioning and deprovisioning
7. [Audit & Compliance](markdown/-IAMADM-07-audit-compliance.md) — Auditing access and ensuring compliance
8. [Multi-Environment](markdown/-IAMADM-08-multi-environment.md) — IAM across multiple environments
9. [Troubleshooting](markdown/-IAMADM-09-troubleshooting.md) — Diagnosing and resolving IAM issues

## Usage
1. Choose a format: import JSON from NOTEBOOKS/, read PDFs/ for print, or view markdown/ for lightweight browsing.
2. Start with Governance Foundations, then progress through the series based on your needs.
